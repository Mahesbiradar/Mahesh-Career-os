## Section 5 — Python Topics (Detailed)

### Topic 1: Variables, Data Types, Type System

**What to understand (ML-3 target):**
- Python's dynamic typing (variable type is determined at assignment, not declaration)
- All primitive types: int, float, str, bool, None, bytes
- Type checking: `type()`, `isinstance()`
- Type conversion: `int()`, `str()`, `float()`, `bool()`
- Mutability: which types are mutable (list, dict, set) vs immutable (int, str, tuple)
- Why mutability matters (a concept that will be critical for OOP later)

**Mini-task (ML-3 evidence):**
Write a Python script that:
1. Declares variables of every primitive type
2. Demonstrates type conversion (str → int, int → float, bool → int)
3. Shows `type()` and `isinstance()` in use
4. Includes a comment (one line) that demonstrates understanding of mutability

**Quick Self-Check (ML-3):**
> "Without looking anything up: what is the difference between mutable and immutable? Name one mutable and one immutable type. What happens when you convert `bool(0)`, `bool("")`, `bool([])` — and why?"

If this question cannot be answered fluently: topic is at ML-2. Do not advance to Topic 2 until ML-3 is reached.

---

### Topic 2: Operators

**What to understand (ML-3 target):**
- Arithmetic: `+`, `-`, `*`, `/`, `//` (floor division), `%` (modulo), `**` (power)
- Comparison: `==`, `!=`, `<`, `>`, `<=`, `>=`
- Logical: `and`, `or`, `not` (short-circuit evaluation)
- Assignment: `=`, `+=`, `-=`, `*=`, `/=`
- Membership: `in`, `not in`
- Identity: `is`, `is not` (why `is` ≠ `==`)
- Operator precedence basics

**Mini-task (ML-3 evidence):**
Write a script that:
1. Demonstrates floor division vs regular division (show the difference)
2. Shows short-circuit evaluation (`and`, `or`) with print statements proving it
3. Demonstrates the difference between `==` and `is` (use small integers and large integers to show where `is` behaves unexpectedly)

**Quick Self-Check (ML-3):**
> "What does `7 // 2` return? What does `7 % 2` return? What is short-circuit evaluation and why does it matter? What is the difference between `==` and `is`?"

---

### Topic 3: Strings

**What to understand (ML-3 target):**
- String immutability (you cannot modify a string in place — you create a new one)
- f-strings (the modern format — `f"Hello {name}"`)
- Common string methods: `.upper()`, `.lower()`, `.strip()`, `.split()`, `.join()`, `.replace()`, `.find()`, `.startswith()`, `.endswith()`, `.count()`
- String slicing: `s[start:stop:step]`
- Multi-line strings: triple quotes
- String multiplication and concatenation

**Mini-task (ML-3 evidence):**
Write a script that:
1. Takes a sample sentence and uses 8+ string methods on it (different methods, not the same one 8 times)
2. Demonstrates slicing: extract the first 5 chars, the last 3 chars, reverse a string
3. Uses f-strings to build a formatted output (not `+` concatenation, not `.format()`)
4. Shows why `"hello" + 5` raises a TypeError and how to fix it

**Quick Self-Check (ML-3):**
> "If `s = 'Hello World'`, what does `s[:5]`, `s[-5:]`, and `s[::2]` return? What is the difference between `.split()` and `.join()`? Why can't you do `s[0] = 'J'`?"

### Topic 4: Lists

**What to understand (ML-3 target):**
- List creation, indexing (positive and negative), slicing
- Mutability: lists can be modified in place
- Common methods: `.append()`, `.extend()`, `.insert()`, `.remove()`, `.pop()`, `.sort()`, `.reverse()`, `.index()`, `.count()`, `.copy()`
- List unpacking: `a, b, c = [1, 2, 3]`
- Nested lists
- Difference between `.copy()` (shallow) and no copy (reference)

**Mini-task (ML-3 evidence):**
Write a script that:
1. Creates a list of 5 items
2. Demonstrates 8 different list methods with results printed
3. Shows the difference between `list_a = list_b` (reference) vs `list_a = list_b.copy()` (copy) by modifying one and printing both
4. Demonstrates negative indexing

**Quick Self-Check (ML-3):**
> "What does `list.pop()` return? What is the difference between `.remove()` and `.pop()`? If `a = [1, 2, 3]` and `b = a` and you do `b.append(4)`, what does `a` now contain and why?"

### Topic 5: Tuples

**What to understand (ML-2 target):**
- Tuple creation: `(1, 2, 3)` and the single-item gotcha `(1,)`
- Immutability: why tuples cannot be modified
- When to use tuple over list (when data should not change)
- Tuple packing and unpacking: `a, b = (1, 2)`
- Tuples as dictionary keys (because they're hashable)
- `tuple()` conversion from list

**Mini-task (ML-2 evidence):**
Write a script that:

**Quick Self-Check (ML-2):**

### Topic 6: Dictionaries

**What to understand (ML-3 target):**
- Dictionary creation: `{}` and `dict()`
- Key-value access: `d[key]`, `d.get(key)`, `d.get(key, default)`
- Mutation: add, update, delete keys
- Common methods: `.keys()`, `.values()`, `.items()`, `.update()`, `.pop()`, `.get()`, `.setdefault()`
- Iteration: `for k, v in d.items()`
- Nested dictionaries
- Dict comprehension (basic): `{k: v for k, v in pairs}`

**Mini-task (ML-3 evidence):**
Write a script that:
1. Creates a dictionary of ≥ 5 items
2. Demonstrates `.get()` with a default value (show why this is safer than `d[key]`)
3. Iterates over `.items()` and prints formatted key-value pairs
4. Writes one dict comprehension (e.g., squares of numbers)
5. Demonstrates nested dictionary access

**Quick Self-Check (ML-3):**
> "What is the difference between `d['key']` and `d.get('key')`? What does `d.items()` return? Write a dict comprehension that maps numbers 1–5 to their squares. What happens when you try to access a key that doesn't exist?"

### Topic 7: Sets

**What to understand (ML-2 target):**
- Set creation: `{1, 2, 3}` (note: `{}` creates a dict, not a set — use `set()`)
- Sets are unordered and contain no duplicates
- Set operations: union (`|`), intersection (`&`), difference (`-`), symmetric difference (`^`)
- Membership testing: `in` (O(1) — faster than list)
- `.add()`, `.remove()`, `.discard()`
- When to use sets: deduplication, fast membership checks

**Mini-task (ML-2 evidence):**
Write a script that:
1. Creates two sets
2. Demonstrates all 4 set operations with outputs
3. Shows deduplication: `list(set([1, 2, 2, 3, 3, 3]))` and explains why it works
4. Compares `in` membership check on a list of 10,000 items vs a set (or just explains the O(1) vs O(n) difference in a comment)

**Quick Self-Check (ML-2):**
> "How do you create an empty set? What is the output of `{1, 2, 3} & {2, 3, 4}`? Why are sets faster than lists for membership testing?"

### Topic 8: Conditionals and Loops

**What to understand (ML-3 target):**
- `if / elif / else` — correct syntax, indentation rules
- Ternary expression: `value = x if condition else y`
- `for` loop — iterating over list, dict, range, string
- `while` loop — condition-based iteration
- `break`, `continue`, `pass`
- `else` on loops (rarely used but interview relevant)
- `enumerate()` — loop with index
- `zip()` — iterate two sequences together
- Nested loops and when to avoid them

**Mini-task (ML-3 evidence):**

3. A `while` loop with `break` on a condition

**Quick Self-Check (ML-3):**
> "Without looking it up, write the FizzBuzz loop for 1–15 in your head. What does `enumerate(['a', 'b', 'c'])` produce? What is `for ... else` and when would you use it?"

## Section 6 — Backend Topics (Week 1)

Backend in Week 1 is a mental model exercise. No code. No framework. No setup.

**Goal:** By end of Week 1, Mahesh can close all references and write 5 short paragraphs explaining these concepts in plain English.

### Backend Conceptual Topics

**Topic B-1: The HTTP Request/Response Cycle**

Understand the complete lifecycle of a web request:
1. User types URL in browser
2. DNS resolves hostname to IP address
3. TCP connection established (3-way handshake)
4. HTTP request sent (method + path + headers + optional body)
5. Server processes request
6. HTTP response returned (status code + headers + body)
7. Browser renders the response

Evidence: One paragraph (150+ words) explaining this flow in Mahesh's own words, written from memory with no reference.

**Topic B-2: HTTP Methods**

| Method | Purpose | Body? | Idempotent? |
|--------|---------|-------|------------|
| GET | Retrieve resource | No | Yes |
| POST | Create resource | Yes | No |
| PUT | Replace resource | Yes | Yes |
| PATCH | Partial update | Yes | No |
| DELETE | Remove resource | No | Yes |

Evidence: One paragraph explaining why idempotency matters (if you call PUT twice, the result should be the same as calling it once — why does this matter for API design?).

**Topic B-3: HTTP Status Codes (Top 10)**

Must know without reference:
- 200 OK, 201 Created, 204 No Content
- 301 Moved Permanently, 302 Found
- 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 422 Unprocessable Entity
- 500 Internal Server Error, 503 Service Unavailable

Evidence: Write each code + meaning in the notes file. Then close the file. Can you recall all 12 without looking?

**Topic B-4: URL Anatomy**
`https://api.example.com:8000/users/123?format=json#section`
- Scheme: `https`
- Host: `api.example.com`
- Port: `8000`
- Path: `/users/123`
- Query string: `?format=json`
- Fragment: `#section`

Evidence: Draw or write this breakdown in the notes file with a real API URL from an existing project.

**Topic B-5: What is a Web Framework?**

In Mahesh's own words: what problem does a web framework solve? Why would a developer use Django instead of writing everything from scratch? What does Django provide that raw Python does not?

Evidence: One paragraph (100+ words). This paragraph will be used verbatim in future interviews when asked "why did you choose Django?"

## Section 12 — Failure Recovery Rules

### Week 1-Specific Recovery Rules

**WFR-1: Missed Day in Week 1**
Week 1 has no streak to protect yet. A missed day is a missed day — not a crisis.
- File a zero-log entry with the reason
- Do not add extra hours to compensate the next day
- Resume normally on the next study day
- If 3 or more days are missed: Week 1 extends by those days; Week 2 does not start until 5 study days are complete

**WFR-2: Topic Not Reaching ML-3 After Two Sessions**
- Do not advance to the next topic
- Reduce scope: focus on the single part that is failing, not the whole topic
- Example: If dictionary iteration is stuck but dict creation is fine — work only on `.items()` iteration for 30 focused minutes
- File an honest log with ADS reflecting how much AI help was needed
- ML-2 at the end of Week 1 is not failure — it is honest progress

**WFR-3: Implementation Not Working After 60 Minutes**
- Reduce the mini-task to its simplest component
- If the simple component also fails: the study phase (Block 1) was insufficient — re-read the relevant doc section and try again
- Do not ask AI to fix the implementation
- Do commit the broken attempt to GitHub with a message that describes what was tried: `"attempt: dict comprehension — broken, iteration order issue"`
- A committed broken attempt is evidence of effort and honest struggle. It is valuable.

**WFR-4: Motivation Low in Week 1**
Week 1 motivation problems are almost always caused by one of three things:
1. **Environment friction** (Python not set up smoothly, Git config issues): Fix the environment before studying anything else
2. **Topic too abstract** (data types feel pointless): Build something small with them — a name formatter, a simple calculator — and the relevance becomes concrete
3. **Comparison to others**: There is no comparison. Week 1 is building the machine. The machine will be invisible to everyone else until Month 3.

**WFR-5: Communication Practice Skipped**
This happens. The 15 minutes feels like an interruption.
- Do not skip it two days in a row
- If motivation to speak aloud is low: write instead. Write a new version of the self-introduction, then read it aloud once
- The minimum viable communication session is 5 minutes: deliver the self-introduction twice and do the pronunciation drill once
